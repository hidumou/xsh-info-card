# Topic JSON Schema

每期话题的结构化数据。五张卡片共用一份 JSON，字段按卡片类型分区。

## 顶层结构

```jsonc
{
  "slug": "2026-04-23-huoermuzi-boyi",      // 必填：文件夹名 / 编号，支持 ASCII 或中文
  "meta": { ... },                           // 元信息，所有卡片共享
  "cover":     { ... },                      // 1-封面
  "abstract":  { ... },                      // 2-摘要
  "takeaways": { ... },                      // 3-要点
  "quote":     { ... },                      // 4-金句
  "further":   { ... }                       // 5-延伸阅读
}
```

如果某张卡不想生成，省略对应字段即可（生成器会跳过）。

## `meta` — 全局元信息

| 字段 | 必填 | 说明 |
|---|---|---|
| `issue_no` | ✅ | 期号字符串，如 `"019"` |
| `date` | ✅ | 日期 `"2026.04.23"` 或 `"2026-04-23"` |
| `topic_tag` | ✅ | 主题标签，如 `"地缘风险 · MIDDLE EAST × OIL"` |
| `rating` | ☐ | 推荐度，如 `"★★ 必读 · MUST READ"` |
| `kicker_default` | ☐ | 每张卡 kicker 的默认值，如 `"DAILY BRIEF · 今日信号"` |

## `cover` — 封面

| 字段 | 必填 | 说明 |
|---|---|---|
| `title` | ✅ | 主标题文本，支持 `{em}...{/em}` 高亮标记 |
| `dek` | ✅ | 导语，一两句说明 |
| `source_name` | ✅ | 信号来源名 |
| `source_meta` | ☐ | 来源补充（作者/日期/交叉佐证） |
| `kpi` | ☐ | 三栏数据条，每条 `{"key":"BRENT","value":"$100+","tone":"default\|down\|up"}` |

## `abstract` — 摘要

| 字段 | 必填 | 说明 |
|---|---|---|
| `title` | ☐ | 默认复用 cover.title |
| `byline` | ✅ | 作者/来源注记，支持简单 HTML（`<strong>` 等） |
| `abstract` | ✅ | 摘要正文，支持 `{em}` 标记 |
| `editor_note` | ✅ | 编者按正文 |

## `takeaways` — 三点速读

| 字段 | 必填 | 说明 |
|---|---|---|
| `title` | ✅ | 大标题，支持 `{em}` |
| `items` | ✅ | 3 条，每条 `{"title":"…","body":"…","src":"…"}`，`src` 可选 |

## `quote` — 金句

| 字段 | 必填 | 说明 |
|---|---|---|
| `text` | ✅ | 金句正文，支持 `{em}` |
| `cite` | ✅ | 署名 |
| `context` | ☐ | 补充说明（一行小字） |

## `further` — 延伸阅读

| 字段 | 必填 | 说明 |
|---|---|---|
| `title` | ✅ | 大标题 |
| `groups` | ✅ | 分组列表 `[{"title":"…","items":[{"text":"…","src":"…"}, ...]}, ...]` |

## 高亮标记约定

正文字段里可以用 `{em}关键词{/em}` 标记需要高亮的短语，渲染为 HTML `<em>` —— 每个风格的 CSS 会自动把 `<em>` 按本模板主色（金色/朱砂红/信号绿等）高亮。

换行用 `<br>` 直接写在字段值里即可，会原样输出（Jinja2 设为不转义）。
